* openapi.exe

: inspect and transform OpenAPI specifications
:
:   openapi.exe SUBCOMMAND
:
: === subcommands ===
:
:   generate                   . generate an OCaml library into a directory
:   init                       . initialize a new OpenAPI-generated library
:   inspect                    . parse and display summarized information about a
:                                spec
:   version                    . print version information
:   help                       . explain a given subcommand (perhaps recursively)

** version

: print version information
:
:   openapi.exe version
:
: === flags ===
:
:   [-build-info]              . print build info for this build
:   [-version]                 . print the version of this build
:   [-help], -?                . print this help text and exit

** generate

: generate an OCaml library into a directory
:
:   openapi.exe generate
:
: === flags ===
:
:   -name STRING               . project name
:   -out FILE                  . the output directory
:   -spec FILE                 . a v3.0.3 spec file
:   [-generated-files-archive FILE]
:                              . By default, the generator will output all the ml
:                                files as they are in the directory. However, that
:                                complicated build rules when the schema or code
:                                generator are updated, so if you pass in this
:                                argument, the ml files will be archived to the
:                                given filename instead.
:   [-include-unknown-fallback-for-enums]
:                              . Include an unknown fallback for enums instead of
:                                raising if an unexpected value is found
:   [-raise-on-optional-null]  . Use the legacy behavior of raising if an optional
:                                field contains null
:   [-help], -?                . print this help text and exit

** init

: initialize a new OpenAPI-generated library
:
:   openapi.exe init
:
: === flags ===
:
:   -build-data-app STRING     . build-data app name
:   -build-data-filename STRING
:                              . schema filename in build-data
:   -build-data-version STRING . build-data version
:   -name STRING               . library name
:   -out DIR                   . output directory
:   -spec FILE                 . path to spec file (relative to output dir)
:   [-include-unknown-fallback-for-enums]
:                              . Include an unknown fallback for enums instead of
:                                raising if an unexpected value is found
:   [-raise-on-optional-null]  . Use the legacy behavior of raising if an optional
:                                field contains null
:   [-help], -?                . print this help text and exit

:   openapi.exe inspect SUBCOMMAND

: === subcommands ===

:   help                       . explain a given subcommand (perhaps recursively)
